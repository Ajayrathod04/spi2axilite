# Hardware Pin Mapping & Signal Descriptions

This document outlines the pinout and virtual signal mappings for both the external serial SPI port and the internal parallel AXI4-Lite bus.

---

## 1. SPI Interface Port Map (External Board Wires)

These signals connect the FPGA bridge to the external host microcontroller:

| Signal Name | Direction | Clock Domain | Width | Engineering & Educational Explanation |
|---|---|---|---|---|
| **`mosi`** | Input | `sclk` | 1-bit | **Master Output Slave Input:** The serial data wire where the external master shifts out bits. The bridge samples this line synchronously on the rising edge of `sclk`. |
| **`miso`** | Output | `sclk` | 1-bit | **Master Input Slave Output:** The serial data wire where the bridge shifts out read response bytes back to the master. Bits are driven on the falling edge of `sclk` and the line is tri-stated (`high-z`) when `cs_n` is inactive high. |
| **`sclk`** | Input | Asynchronous | 1-bit | **Serial Clock:** The clock signal generated by the external master to coordinate transmission. The serial transfer rate is independent of the FPGA system clock. |
| **`cs_n`** | Input | Asynchronous | 1-bit | **Chip Select:** Active-low select line. Pulling `cs_n` low wakes up the slave and starts the packet. Deasserting `cs_n` high immediately aborts FSM activity and resets the bridge back to `IDLE` within one system clock cycle. |

---

## 2. AXI4-Lite Internal Bus Port Map (Virtual System Wires)

These signals connect the internal `axi_lite_master` to the `axi_register_bank` inside the FPGA:

### Write Address Channel
| Signal Name | Direction | Clock Domain | Width | Engineering & Educational Explanation |
|---|---|---|---|---|
| **`awaddr`** | Master to Slave | `clk` | 32-bit | **Write Address:** Drive the 32-bit target register address to write data into. |
| **`awvalid`** | Master to Slave | `clk` | 1-bit | **Write Address Valid:** High-active flag indicating the master is driving a valid write address. |
| **`awready`** | Slave to Master | `clk` | 1-bit | **Write Address Ready:** High-active response from the register bank indicating it is ready to accept the address. |

### Write Data Channel
| Signal Name | Direction | Clock Domain | Width | Engineering & Educational Explanation |
|---|---|---|---|---|
| **`wdata`** | Master to Slave | `clk` | 32-bit | **Write Data:** Drive the 32-bit parallel write data payload onto the bus. |
| **`wvalid`** | Master to Slave | `clk` | 1-bit | **Write Data Valid:** High-active flag indicating the master is driving valid write data. |
| **`wready`** | Slave to Master | `clk` | 1-bit | **Write Data Ready:** High-active response from the register bank indicating it is ready to accept the write data. |

### Write Response Channel
| Signal Name | Direction | Clock Domain | Width | Engineering & Educational Explanation |
|---|---|---|---|---|
| **`bresp`** | Slave to Master | `clk` | 2-bit | **Write Response:** Status of the write transaction (tied to `2'b00` representing `OKAY`). |
| **`bvalid`** | Slave to Master | `clk` | 1-bit | **Write Response Valid:** High-active flag indicating the slave is driving a valid write response. |
| **`bready`** | Master to Slave | `clk` | 1-bit | **Write Response Ready:** High-active response from the master indicating it is ready to accept the write status. |

### Read Address Channel
| Signal Name | Direction | Clock Domain | Width | Engineering & Educational Explanation |
|---|---|---|---|---|
| **`araddr`** | Master to Slave | `clk` | 32-bit | **Read Address:** Drive the 32-bit target register address to read data from. |
| **`arvalid`** | Master to Slave | `clk` | 1-bit | **Read Address Valid:** High-active flag indicating the master is driving a valid read address. |
| **`arready`** | Slave to Master | `clk` | 1-bit | **Read Address Ready:** High-active response from the register bank indicating it is ready to accept the read address. |

### Read Data Channel
| Signal Name | Direction | Clock Domain | Width | Engineering & Educational Explanation |
|---|---|---|---|---|
| **`rdata`** | Slave to Master | `clk` | 32-bit | **Read Data:** Drive the 32-bit parallel read data payload from the target register onto the bus. |
| **`rvalid`** | Slave to Master | `clk` | 1-bit | **Read Data Valid:** High-active flag indicating the slave is driving valid read data. |
| **`rready`** | Master to Slave | `clk` | 1-bit | **Read Data Ready:** High-active response from the master indicating it is ready to accept the read data. |
| **`rresp`** | Slave to Master | `clk` | 2-bit | **Read Response:** Status of the read transaction (tied to `2'b00` representing `OKAY`). |
