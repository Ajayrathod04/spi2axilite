# Hardware Pin Mapping & Signal Interface

This document contains a comprehensive pin mapping table and signal interface list for the **SPI to AXI4-Lite Bridge**, outlining the external SPI connections and internal AXI4-Lite bus channels.

---

## 1. External SPI Hardware Pins (Physical Layer)

These pins connect the bridge to the external SPI Master (host MCU/DSP/FPGA):

| Pin Name | Direction | Active Level | Width | Protocol | Signal Description |
|---|---|---|---|---|---|
| **`mosi`** | Input | High | 1-bit | SPI | **Master Out Slave In**: Serial data driven by the master and sampled on the rising edge of `sclk`. |
| **`miso`** | Output | High / Tri-state | 1-bit | SPI | **Master In Slave Out**: Serial data driven by the bridge on the falling edge of `sclk`. High-impedance (`z`) when `cs_n` is inactive. |
| **`sclk`** | Input | Rising Edge | 1-bit | SPI | **SPI Serial Clock**: Synchronous timing clock generated by the external master (SPI Mode 0, CPOL=0). |
| **`cs_n`** | Input | Active-Low | 1-bit | SPI | **Chip Select / Enable**: Active-low line indicating the start of a 24-bit SPI packet. |

---

## 2. Internal AXI4-Lite Bus Interface (Bus Layer)

These virtual bus lines connect the `axi_lite_master` submodule to the `axi_register_bank` (AXI Slave) within the chip:

### A. Write Address Channel
| Signal Name | Direction | Active Level | Width | Signal Description |
|---|---|---|---|---|
| `awaddr` | Master $\rightarrow$ Slave | High | 32-bit | **Write Address**: Target register address for the write operation. |
| `awvalid` | Master $\rightarrow$ Slave | High | 1-bit | **Write Address Valid**: Indicates the write address is stable and ready. |
| `awready` | Slave $\rightarrow$ Master | High | 1-bit | **Write Address Ready**: Indicates the register bank can accept the address. |

### B. Write Data Channel
| Signal Name | Direction | Active Level | Width | Signal Description |
|---|---|---|---|---|
| `wdata` | Master $\rightarrow$ Slave | High | 32-bit | **Write Data**: 32-bit data to be written (data mapped to `wdata[7:0]`). |
| `wstrb` | Master $\rightarrow$ Slave | High | 4-bit | **Write Strobe**: Byte lane select. Set to `4'hF` (all bytes enabled). |
| `wvalid` | Master $\rightarrow$ Slave | High | 1-bit | **Write Data Valid**: Indicates the write data is stable and ready. |
| `wready` | Slave $\rightarrow$ Master | High | 1-bit | **Write Data Ready**: Indicates the register bank can accept the data. |

### C. Write Response Channel
| Signal Name | Direction | Active Level | Width | Signal Description |
|---|---|---|---|---|
| `bresp` | Slave $\rightarrow$ Master | High | 2-bit | **Write Response**: Status of write (`2'b00` = OKAY). |
| `bvalid` | Slave $\rightarrow$ Master | High | 1-bit | **Write Response Valid**: Indicates the response is stable and ready. |
| `bready` | Master $\rightarrow$ Slave | High | 1-bit | **Write Response Ready**: Indicates the master can accept the write status. |

### D. Read Address Channel
| Signal Name | Direction | Active Level | Width | Signal Description |
|---|---|---|---|---|
| `araddr` | Master $\rightarrow$ Slave | High | 32-bit | **Read Address**: Target register address for the read operation. |
| `arvalid` | Master $\rightarrow$ Slave | High | 1-bit | **Read Address Valid**: Indicates the read address is stable and ready. |
| `arready` | Slave $\rightarrow$ Master | High | 1-bit | **Read Address Ready**: Indicates the register bank can accept the address. |

### E. Read Data Channel
| Signal Name | Direction | Active Level | Width | Signal Description |
|---|---|---|---|---|
| `rdata` | Slave $\rightarrow$ Master | High | 32-bit | **Read Data**: 32-bit data read from the target register. |
| `rresp` | Slave $\rightarrow$ Master | High | 2-bit | **Read Response**: Status of read (`2'b00` = OKAY). |
| `rvalid` | Slave $\rightarrow$ Master | High | 1-bit | **Read Data Valid**: Indicates the read data is stable and ready. |
| `rready` | Master $\rightarrow$ Slave | High | 1-bit | **Read Data Ready**: Indicates the master can accept the read data. |

---

## 3. Register Memory Map

The AXI Register Bank decodes these addresses in the address space:

| AXI Address | Register Name | Bit Range | Access | Default Value | Hardware Description |
|---|---|---|---|---|---|
| **`32'h0000_0000`** | **CONTROL** | `[31:0]` | R/W | `32'h0000_0000` | System control. |
| **`32'h0000_0004`** | **STATUS** | `[31:0]` | R-Only | `32'h0000_0001` | Reads system status (`1` = active/ready). |
| **`32'h0000_0008`** | **DATA0** | `[31:0]` | R/W | `32'h0000_0000` | Data register 0 (general purpose). |
| **`32'h0000_000C`** | **DATA1** | `[31:0]` | R/W | `32'h0000_0000` | Data register 1 (general purpose). |
