# Protocol Implementation Notes

This document provides a technical reference on the **SPI** and **AXI4-Lite** protocols as implemented in the bridge for the **Write a HDL code for SPI(master) to AXI4-Lite** project, focusing on timing correctness, signal constraints, and structural behaviors.

---

## 1. SPI Protocol implementation (Mode 0)

### Configurations
* **CPOL = 0**: The SPI Clock (`sclk`) is low when idle.
* **CPHA = 0**: The data bits are sampled on the **rising/leading edge** of `sclk` and shifted out/updated on the **falling/trailing edge** of `sclk`.
* **Character Length**: Fixed 8-bit characters (bytes), MSB-first.
* **Chip Select (`cs_n`)**: Active-low. Indicates the start (falling edge) and end (rising edge) of a SPI packet.

### Timing Diagram (SPI Mode 0)
```
            __                                                         __
CS_N  \____/  \_______________________________________________________/  \___
              _    _    _    _    _    _    _    _    _    _    _    _
SCLK  _______/ \__/ \__/ \__/ \__/ \__/ \__/ \__/ \__/ \__/ \__/ \__/ \______
             :    :    :    :    :    :    :    :    :    :    :    :
MOSI  ======X MSB X B6 X B5 X B4 X B3 X B2 X B1 X LSB X======================
             ^    ^    ^    ^    ^    ^    ^    ^    ^    ^    ^    ^
             |    |    |    |    |    |    |    |    |    |    |    |
           Sample Sample...                                                 (Rising Edges)
MISO  ======X MSB X B6 X B5 X B4 X B3 X B2 X B1 X LSB X======================
```

### Key Timing Rules
1. **MOSI Setup**: The master must drive the first bit (MSB) onto `mosi` as soon as `cs_n` goes low, before the first rising edge of `sclk`. For each subsequent bit, the master drives MOSI on the falling edge of `sclk`.
2. **MISO Setup**: The slave must drive the first bit of its response onto `miso` as soon as the response state (`SEND_RESP`) is entered (specifically on the 16th falling edge of `sclk`), allowing the master to sample it on the 17th rising edge of `sclk`.

## 2. AXI4-Lite Protocol Implementation

AXI4-Lite is a simplified, point-to-point sub-set of the AXI4 interface, designed for register-style access. 

### Key Features Implemented:
* **32-Bit Address & 32-Bit Data Channels**: Standard bus sizing.
* **Fully Synchronous Handshakes**: Every channel uses `valid` and `ready` handshake lines. Data is transferred only when both `valid` and `ready` are high on a rising edge of `clk`.
* **No Burst Transfers**: Every read/write is exactly one word. `axlen` and `axburst` are omitted.
* **No Complex Pipelining**: Transactions are completed sequentially.

### Register Address Map:
All addresses are byte-aligned. Since registers are 32-bit (4-byte), they are spaced by 4 bytes:

| SPI ADDR | AXI Address | Register Name | Access Type | Default Value | Description |
|---|---|---|---|---|---|
| `8'h00` | `32'h00000000` | **CONTROL** | Read/Write | `32'h00000000` | Configures bridge behavior / soft resets. |
| `8'h04` | `32'h00000004` | **STATUS** | Read-Only | `32'h00000001` | Returns status flags (e.g. 1 = Active/Ready). |
| `8'h08` | `32'h00000008` | **DATA0** | Read/Write | `32'h00000000` | User data register 0. |
| `8'h0C` | `32'h0000000C` | **DATA1** | Read/Write | `32'h00000000` | User data register 1. |

## 3. Clocking and Metastability Mitigation

Since `sclk`, `cs_n`, and `mosi` are generated by an external SPI Master, they are asynchronous to the internal system clock `clk`. 

Directly sampling these inputs in the sequential FSM would cause metastability and functional errors. To mitigate this:
1. **Synchronizer Chains**: All inputs from the SPI master (`sclk`, `cs_n`, `mosi`) pass through **2-stage flip-flop synchronizers** clocked by the high-speed system `clk`.
2. **Edge Detectors**: We use a 3rd stage register to compare the current synchronized value with its previous state, cleanly detecting rising (`01`) and falling (`10`) edges of `sclk` and `cs_n` synchronously inside the system clock domain.
3. **FSM Independence**: The system clock `clk` runs significantly faster than the SPI clock `sclk` (typically 10-100x). This speed delta gives the AXI4-Lite bus ample clock cycles to complete read/write operations during the SPI clock periods, ensuring zero-latency data availability for SPI responses.
